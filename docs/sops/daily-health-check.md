# Daily Windows Server Health Check SOP

## Overview

| Field | Value |
|---|---|
| **Frequency** | 4× per day: 06:00, 12:00, 18:00, 00:00 UTC |
| **Scope** | All Azure Arc-enrolled Windows servers |
| **Automation tier** | Tier 1 (Azure Functions) + Tier 2 (AI insight) |
| **Owner** | Wintel SRE team |

---

## Checks Performed

### 1. Disk Usage

Query each drive on the server.

| Threshold | Action |
|---|---|
| > 80% used | WARNING — AI insight generated |
| > 90% used | CRITICAL — SRE Agent triggered |

### 2. CPU Usage (5-minute average)

Queried via Log Analytics KQL (`Perf` table, `% Processor Time`).

| Threshold | Action |
|---|---|
| > 80% avg | WARNING |
| > 95% avg | CRITICAL |

### 3. Memory Usage

Queried via Log Analytics KQL (`Perf` table, `% Committed Bytes In Use`).

| Threshold | Action |
|---|---|
| > 85% | WARNING |
| > 95% | CRITICAL |

### 4. Windows Services

Critical services that must be running:

| Service Name | Description |
|---|---|
| `wuauserv` | Windows Update |
| `WinRM` | Windows Remote Management |
| `EventLog` | Windows Event Log |
| `MpsSvc` | Windows Firewall |
| `MdCoreSvc` | Microsoft Defender Core Service |

Any service in a non-Running state raises a WARNING. If `MdCoreSvc` or `WinRM` is stopped, escalate to CRITICAL.

### 5. Windows Event Log Errors

Queries System and Application event logs for Error/Critical events in the last 6 hours (Event IDs 1000–9999).

| Count | Action |
|---|---|
| 1–10 errors | WARNING |
| > 10 errors | CRITICAL |

### 6. Server Heartbeat (Arc Connectivity)

Checks `lastStatusChange` timestamp on the Arc machine resource.

| Last Seen | Action |
|---|---|
| > 10 minutes ago | WARNING |
| > 30 minutes ago | CRITICAL — P1 incident raised |

---

## Memory Integration

Before evaluating any threshold, the engine checks:
- **Active suppressions** — if a server is in a maintenance window, suppress all alerts.
- **Threshold overrides** — if a per-server or per-check override exists, apply it instead of the default.

---

## Output

- **Cosmos DB** — `health-runs` container; one document per run with all check results.
- **Portal summary card** — aggregated status displayed in the ops portal.

---

## Escalation Matrix

| Result | Action |
|---|---|
| WARNING (any check) | AI insight generated by Health Analyst Agent |
| CRITICAL (any check) | SRE Agent triggered via webhook |
| Heartbeat lost (>30 min) | P1 incident created in ITSM automatically |
